# LFS148 – Open Source Development Practice

## Overview
This repository contains my completed exercises and learning notes from  
**LFS148 / LFS148x – Getting Started with Open Source Development** by The Linux Foundation.

The goal of this work is not just to complete labs, but to understand how
real-world open source projects are structured, developed, and maintained.

---

## Why This Repository Matters

This course and repository helped me learn how professional software engineers:

- Navigate and understand unfamiliar codebases
- Use Git and GitHub for collaboration
- Work with containerized development environments
- Follow open source contribution workflows
- Document their work clearly for others

This reflects the kind of skills used in real engineering teams.

---

## What I Practiced

- ✅ Git fundamentals (clone, fork, branch, commit)
- ✅ Reading multi-language repositories
- ✅ Python, Java, HTML, and Docker basics
- ✅ Devcontainers and reproducible environments
- ✅ Understanding CI workflows
- ✅ Writing and following documentation

---

## Repository Structure

- `exercises/` – Hands-on course labs and solutions
- `.devcontainer/` – Development container configuration
- `.github/workflows/` – CI workflows
- `assets/` – Supporting assets for exercises
- `.gitpod.yml` – Cloud development environment setup

---

## Learning Reflections

Through this course, I learned that:
- Good documentation is as important as good code
- Small, clean commits tell a story
- Containers reduce “it works on my machine” problems
- Open source is about collaboration, not perfection

This repository represents my first steps into that culture.
