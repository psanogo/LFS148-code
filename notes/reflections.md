# Learning Reflections – LFS148

This file captures what I learned while working through the LFS148 exercises.
The focus is not just on tools, but on how open source development works in practice.

---

## What Felt Confusing at First

- Navigating a repository with many folders and files felt overwhelming
- Understanding how different tools (Git, Docker, CI) fit together was unclear
- I underestimated the importance of documentation before writing code

---

## What Became Clear Over Time

- Large codebases become manageable once you learn how to explore them
- Containers make development environments predictable and repeatable
- Git history tells a story if commits are small and intentional
- Clear documentation helps both others and your future self

---

## Key Skills I Practiced

- Working with forks and upstream repositories
- Reading and understanding code written by others
- Following structured lab instructions independently
- Using dev containers instead of relying on local setup
- Thinking about reproducibility and collaboration

---

## Important Lessons

- Open source is about communication, not perfection
- Asking “why” is more valuable than finishing quickly
- Clean structure and clarity matter as much as functionality
- Learning how to learn is a real engineering skill

---

## How This Changed My Perspective

Before this course, I thought software development was mostly about writing code.
Now I understand it is equally about collaboration, tools, workflows, and clarity.

This repository represents my early steps into professional, open source–style development.
